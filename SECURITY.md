# Security Policy

## Supported Versions

Security updates are provided for the latest code on the default branch.

| Version | Supported |
|---------|-----------|
| Latest default branch | :white_check_mark: |
| Older commits/releases | :x: |

## Reporting a Vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Send a private report with as much detail as possible:
- Description of the issue and potential impact
- Steps to reproduce
- Affected files or components
- Proof of concept (if available)
- Suggested mitigation (optional)

### Contact

- Preferred: GitHub private vulnerability report (Security Advisories)
- Alternate: direct message to the maintainer account

## What to Expect

- Acknowledgement target: within 7 days
- Initial triage target: within 14 days
- Fix timeline: depends on severity and complexity

If the report is accepted, a fix will be prepared and released as soon as practical. Credit can be provided unless you prefer to remain anonymous.

## Scope

This policy applies to this repository and its distributed application code.

Out of scope:
- General support requests
- Third-party service outages or misconfigurations outside this repository
- Known risks explicitly documented as non-goals
